This is the documentation for the app healthy potatos
Endpoints:
Business logic URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of all movies to the user /movies GET NONE A JSON object holding data on all movies
Return a movie to the user /movies/[movie] GET NONE A JSON object holding data on a specific movies
Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[movie]/genre GET NONE Text of the genre based of the movie
Return data about a director (bio, birth year, death year) by name /movies/[movie]/[director] GET NONE A JSON object holding information based off of the director of the movie
Allow new users to register /users POST Json object holding data about the user to add structured like:
{
username: "guest"
password: "password"
email: "email"
dob: "10/25/1980"
}
 Json object holding data about the user that was added:
{
username: "guest"
password: "password"
email: "email"
dob: "10/25/1980"
}
Allow users to update their user info (username, password, email, date of birth) /users PUT none Text message indicating the information was updated
Allow users to add a movie to their list of favorites /users/[username]/[movies] PUT none A text message indicating the movie was successfully added
Allow users to remove a movie from their list of favorites /users/[username]/[movies] DELETE none A text message indicating the movie was successfully removed
Allow existing users to deregister /users/[username] DELETE None A text message indicating the user was successfully removed
